# Introduction

`anvil` generates random, synthesizable SystemVerilog RTL. You give
it a seed and a handful of knobs; it gives you back one or many
SystemVerilog modules that are guaranteed to:

1. **Parse** — valid SystemVerilog syntax.
2. **Elaborate** — valid semantics: widths match, names resolve, no
   multi-driven nets, no illegal lvalues.
3. **Synthesize** — lives inside the subset of SystemVerilog that
   Yosys, Vivado, Design Compiler, and friends actually accept.
4. **Remain structurally non-trivial** — every output genuinely depends
   on at least one real source; the whole circuit does not collapse to a
   constant.

What it emits is random, not meaningful. A gate computing `(a + b) & c`
with no coherent design intent. That is the point: `anvil` is for
**stress-testing tools** (parsers, elaborators, simulators,
synthesizers, formal equivalence checkers) and for **generating
synthetic corpora** — not for synthesizing real designs.

Whole-module intended functionality is usually absent. The target is
legal structure, not behavior with a specification behind it.

The quality bar is intentionally high. `anvil` is not trying to become
"a fuzzer that sometimes emits legal RTL"; it is trying to become a
**signoff-grade random synthesizable RTL generator**. The interesting
part is not malformed input. The interesting part is **legal,
reproducible, unusual RTL** that downstream tools ought to handle
cleanly, but occasionally do not.

## Who is this for?

- **EDA tool developers** who need a steady stream of legal,
  interesting-shaped SV modules to feed to their test infrastructure.
- **Hardware test engineers** who want to exercise their toolchain
  beyond whatever hand-written designs they happen to have on hand.
- **Researchers** studying RTL optimization, synthesis, or
  verification scaling — `anvil`-generated corpora are
  reproducible-by-seed and size-controllable-by-knob.
- **Anyone curious about constrained-random RTL generation** as a
  technique. `anvil` is the reference implementation for
  "generation by construction" applied to synthesizable Verilog.

## Five-minute pitch

```bash
cargo run --release -- --seed 20 --max-depth 2 --max-inputs 2 --max-outputs 1
```

produces

```systemverilog
// Generated by anvil. Module: mod_20_0000
module mod_20_0000 (
    input  logic  clk,
    input  logic  rst_n,
    input  logic [14:0] i_0,
    input  logic [25:0] i_1,
    output logic [24:0] o_0
);

    logic [24:0] flop_0;

    always_ff @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            flop_0 <= 25'h0;
        end else begin
            flop_0 <= flop_0;
        end
    end

    assign o_0 = flop_0;
endmodule
```

That's a minimal hold-register: a 25-bit flop that resets to zero
and holds whatever it's loaded with. The output drives from the
flop's Q. Internal names follow the Rule 12 scheme — flops are
`flop_<id>` and gate wires (not present in this tiny module) would
be `<kind>_<N>`.

Byte-identical output, forever, from the same command. Knobs scale
this from minimal to full-bore sequential logic with DAG-shaped
datapaths, multi-arm muxes, and any flop motif real synchronous
designs use.

See **[Getting Started](getting-started.md)** for install and your
first batch, and **[Tutorial](tutorial.md)** for a progression of
worked examples.

## What makes anvil different

Grammar-based fuzzers (walking an EBNF) produce syntactically valid
text that almost never elaborates — semantic constraints of RTL
(widths, scopes, driver uniqueness, synthesizability subsets) are
far too tight for random grammar walks to satisfy.

Hand-written test suites are tiny and cluster in the corner of the
legal space that the author happened to think of.

`anvil` takes a third path: **build a typed circuit graph by
recursive construction**, where every step is locally constrained to
preserve the invariants that make the output legal. Then pretty-print
SystemVerilog from the graph as a final step. No filter loop. No
generate-then-repair. Validity is structural — the graph *cannot* be
illegal because there was never a moment in its construction when it
could have become illegal.

That by-construction discipline is what lets `anvil` aim higher than a
syntax fuzzer. If a downstream tool breaks, the ideal outcome is that it
breaks on an input that is not merely parseable, but structurally
solid, synthesizable, and boringly acceptable according to the same
standards a signoff flow would expect.

The algorithm is small (a few hundred lines) and easy to extend.
Adding a new motif — a new gate, a new flop style, a new hierarchical
pattern — is a handful of lines in a single function.
